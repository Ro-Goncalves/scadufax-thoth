# Status das ADRs

As skills falam em termos de oito status canônicos de ADR. Este arquivo mapeia esses status para as strings reais usadas nos ADRs deste repositório e define quando cada um deve ser utilizado.

| Status canônico nas skills | Status no nosso repositório | Significado |
| --- | --- | --- |
| Draft | Draft | O arquivo foi criado e está sendo construído durante a conversa. Há contexto suficiente para abrir a ADR, mas a decisão ainda está incompleta. |
| Proposed | Proposed | Já existe uma proposta clara, com contexto, opções e trade-offs mínimos, mas a decisão ainda não foi confirmada. |
| Under Review | Under Review | A proposta foi consolidada e está em revisão, validação ou alinhamento antes da confirmação final. |
| Accepted | Accepted | A decisão foi confirmada explicitamente e passa a ser a decisão vigente. |
| Implemented | Implemented | A decisão já foi aplicada no sistema, no processo ou na arquitetura. |
| Rejected | Rejected | A proposta foi considerada e descartada. Mantém-se por histórico, mas não orienta implementação. |
| Deprecated | Deprecated | A decisão existe historicamente, mas deixou de ser recomendada para novos casos. |
| Superseded | Superseded | A decisão foi substituída por outra ADR mais recente e não é mais a referência vigente. |

## Fluxo recomendado

Draft → Proposed → Under Review → Accepted → Implemented
